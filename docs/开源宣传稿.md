## 一、上周发生的事

如果你还没听说，先简单回顾一下。

2026 年 7 月 12 日，安全研究员 cereblab 逆向分析了 xAI 官方发布的 Grok CLI，发现了一个极其离谱的机制：

**Grok CLI 会在每一轮任务开始前和结束后，各打包一次你当前工作目录的完整状态，然后通过一条独立于模型请求的旁路通道，静默上传到 xAI 的 Google Cloud 仓库。**

更离谱的是，它不只是上传当前仓库的代码。

它还跨目录扫描了用户电脑上的 `~/.claude/` 配置文件，把 Claude Code 的 API Key、Skill 文件、全局 AGENTS 规则全部打包上传了。

作者验证的过程堪称教科书级别：

> "我让 Codex 伪造了一个合成仓库去验证，没想到这个狗东西，不仅把整个合成数据库给搬空了，还直接把我的 .claude 整个配置全部搬走了，连我的 API 密钥都一起传了上去。"

最讽刺的是，当曝光发酵后，xAI 没有站出来解释，而是**通过服务端远程开关，悄悄把上传功能关掉了**。

这意味着两件事：
1. 代码是齐全的，上传管线是完整的
2. 他们随时可以无感重新开启

## 二、更深的恐惧：不只是 Grok 的问题

这篇文章让我后背发凉，不是因为 Grok 够狗，而是因为它揭示了一个**系统性的、行业级的裸奔状态**。

作者原话：

> "Claude Code 能执行你的 Shell 命令，Grok CLI 能扫描你的整个文件系统，Codex 能操控你的浏览器。一个 Agent 产品它们现在拥有的权限，已经跟你电脑上的一个管理员账户没有太大区别了。"
>
> "这个权限量级，在整个软件行业的历史上，只有操作系统和杀毒软件享受过。但操作系统有几十年的安全审计体系，杀毒软件有行业认证和监管框架。AI Agent 有什么？什么都没有。"

**核心问题不是"Grok 在偷数据"，而是"你根本不知道你的 AI 工具在你的电脑上到底在干什么"。**

## 三、我们决定动手验证一下

看完文章后，我们做了两件事。

第一，我们写了一个开源工具，叫 **Moat**（护城河），它有一个命令叫 `moat check --scan-ai`，专门扫描 AI 工具在系统级留下的配置、日志、授权记录。

第二，我们在这台跑着 Claude Code 和 Codex CLI 的 MacBook 上，跑了一次这个命令。

结果让我们自己都吓了一跳。

---

### 🔴 发现 1：Codex CLI — 1.35 GB 的数据

```
🟡 [WARNING] Codex CLI 配置存在: 1354029.7 KB 数据
```

**1.3 GB。** 这是我们 Codex CLI 目录的完整大小，包含所有会话记录、缓存、历史。相当于一个小型数据库的体量。

我们不知道这些数据是否被上传过——因为 Codex 是闭源的，我们无法逆向它的二进制文件。但问题是，**我们也不应该需要逆向二进制才能知道自己的工具在背后做了什么。**

---

### 🟡 发现 2：Claude Code — 23 个敏感命令已授权

```
🟡 [WARNING] Claude Code 已授权 23 个敏感命令
```

当我们打开 `~/.claude/settings.local.json` 时，看到了这些授权记录：

```
⚠️  Bash(sshpass -p 'wj750628' ssh ...)    ← 带密码的 SSH 登录
⚠️  Bash(scp *)                              ← 远程文件传输
⚠️  Bash(tar czf *)                          ← 打包任意文件
⚠️  Bash(curl *)                              ← HTTP 上传/下载
⚠️  Bash(rm -rf packages/...)                 ← 删除文件
⚠️  Bash(chmod +x *)                         ← 授权执行
```

你可能觉得"这些都是我授权过的啊"。但问题是：

**这些授权是永久的、无差别的。** 一旦授权，AI 模型可以在任何上下文中执行这些命令。如果模型被注入恶意指令，这些命令可以直接被用来打包代码、SSH 到远程服务器、甚至通过 curl 上传到任意服务器。

---

### 🟡 发现 3：Claude Code — 302 KB 遥测数据

```
🟡 [WARNING] Claude Code 遥测数据: 24 个文件, 302.3 KB
```

`~/.claude/telemetry/` 目录下有 24 个遥测文件。Anthropic 官方文档没有明确说明这些数据是否会回传。我们不知道——正如我们不知道 Grok 在上传之前一样。

---

### ℹ️ 发现 4：421 KB 的未加密会话历史

```
ℹ️ [INFO] Claude Code 会话历史: 421.9 KB
     📍 /Users/mac/.claude/history.jsonl
```

所有与 AI 的对话记录——包括你在对话中粘贴的代码片段、API 密钥、数据库密码、业务逻辑讨论——都以明文 JSONL 格式存储在硬盘上。任何有本地访问权限的进程都可以读取它们。

## 四、为什么你需要一个"AI 工具审计员"

回顾一下 Grok 事件的时间线：

| 时间 | 事件 |
|------|------|
| 7月10日 | 研究员发现默认上传行为 |
| 7月12日 | 博主公开曝光，帖子迅速传播 |
| 7月13日 | xAI 远程关闭上传功能（但代码管线保留） |

**从发现到关闭，中间隔了 3 天。**

在这 3 天里，有多少用户的代码库被上传了？没人知道。因为没有任何工具可以告诉你——你的 AI 工具在你的电脑上到底在干什么。

这就是为什么我们做了 Moat 的 `moat check --scan-ai`。

它不是一个实时监控器（那是 v2 的事），它是一个**静态审计工具**，可以在一分钟内告诉你：

- 你的机器上有哪些 AI 工具的配置残留
- 这些工具有多少数据存储在本地
- 它们被授权执行了哪些危险命令
- 你的 API Key 和密码是否以明文形式暴露在日志中

## 五、Moat 的立场

我们不想贩卖焦虑。我们想提供一个**你可以自己验证的事实**。

所以 Moat 是开源的：

```bash
pip install moat-ai
moat check --scan-ai
```

就两行命令。你可以在自己的机器上跑，看看结果是什么。

Moat 的定位很简单：

> **AI 时代，你的代码不应该出本地。**

这是我们坚持的 Security Manifesto：

- **Zero-Telemetry**：Moat 不会主动上传任何代码快照、配置文件或 API 密钥
- **Transparent Audit**：Moat 的每一次文件读取都在你的本地审计之下
- **Self-Sovereignty**：你的架构规则、基线数据、记忆索引全部在你控制下
